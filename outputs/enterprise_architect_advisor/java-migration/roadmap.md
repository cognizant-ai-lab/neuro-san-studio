# Roadmap

## Phase 1 — Discovery & Validation
**Key activities:** confirm cloud standard, validate licensing, confirm SOC 2 evidence needs, inventory workloads and dependencies, define migration waves, and assess cutover constraints.  
**Entry criteria:** initial scope known.  
**Exit criteria:** target cloud, migration approach, and risk register approved.  
**Typical duration:** 2-4 weeks.

## Phase 2 — Foundation / Landing Zone
**Key activities:** build landing zone, identity integration, network segmentation, logging/monitoring, key management, backup patterns, and IaC baseline.  
**Entry criteria:** discovery completed.  
**Exit criteria:** cloud foundation ready for workload onboarding.  
**Typical duration:** 2-6 weeks.

## Phase 3 — Core Migration
**Key activities:** migrate Windows and Linux VMs, move SQL Server, validate app runtime, rehearse cutover, execute production cutover, and stabilize.  
**Entry criteria:** landing zone ready and migration waves defined.  
**Exit criteria:** workloads running in cloud with acceptable performance and recovery posture.  
**Typical duration:** 4-12 weeks depending on estate size.

## Phase 4 — Modernisation
**Key activities:** evaluate whether the monolith can be modularized, whether SQL Server should move to a managed service, and whether operational tooling should be standardized further.  
**Entry criteria:** migration stabilized.  
**Exit criteria:** modernization backlog prioritized and approved.  
**Typical duration:** 4-12 weeks for assessment and early improvement backlog creation.

## Phase 5 — Optimisation & AI Enablement
**Key activities:** rightsize VMs, apply reservations or savings plans, refine backup/DR, improve observability, and assess AI/automation opportunities if business demand emerges.  
**Entry criteria:** baseline operations stable.  
**Exit criteria:** cost and reliability improvements embedded into steady state.  
**Typical duration:** ongoing, with first optimization cycle in 4-8 weeks after stabilization.
